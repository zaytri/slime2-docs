An image input can accept either a remote image URL or a local image file.

<Callout type='warning'>
  Local files will only work if they exist in the widget `assets` folder.
</Callout>

If `multiple` is set to `true{:js}`, it can accept multiple image files, and gives you an array of strings.

```ts filename="Setting Definition"
type ImageInputSetting = {
  type: 'image-input'
  label: string
  id: string
  defaultValue?: string | null
  description?: string
}
```

> If no `defaultValue` is given, the default value is `null`.

```ts filename="Setting Definition" {5-6}
type ImageInputSetting = {
  type: 'image-input'
  label: string
  id: string
  multiple: true
  defaultValue?: string[]
  description?: string
}
```

> If no `defaultValue` is given, the default value is an empty array `[]{:js}`.

## Example

```js filename="JavaScript"
addEventListener('slime2:ready', () => {
  slime2.widget.loadSettings('widget-data.js', [
    {
      label: 'Custom Moderator Badge',
      id: 'modBadge',
      type: 'image-input',
      description: 'Replaces the standard green sword badge',
    },
    {
      label: 'Custom VIP Badge',
      id: 'vipBadge',
      type: 'image-input',
      description: 'Replaces the standard purple diamond badge',
    },
  ])
})
```

```json filename="Initial Data"
{ "modBadge": null, "vipBadge": null }
```

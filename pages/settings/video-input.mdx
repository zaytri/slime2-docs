# ðŸ“º Video Input Setting

A video input can accept a local video file.

<Callout type='warning'>
  Local files will only work if they exist in the widget `assets` folder.
</Callout>

If `multiple` is set to `true{:js}`, it can accept multiple video files, and gives you an array of strings.

```ts filename="Setting Definition"
type VideoInputSetting = {
  type: 'video-input'
  label: string
  id: string
  defaultValue?: string | null
  description?: string
}
```

> If no `defaultValue` is given, the default value is `null`.

```ts filename="Setting Definition" {5-6}
type VideoInputSetting = {
  type: 'video-input'
  label: string
  id: string
  multiple: true
  defaultValue?: string[]
  description?: string
}
```

> If no `defaultValue` is given, the default value is an empty array `[]{:js}`.

## Example

```js filename="JavaScript"
addEventListener('slime2:ready', () => {
  slime2.widget.loadSettings('widget-data.js', [
    {
      label: 'Alert Video',
      id: 'alertVideo',
      type: 'video-input',
    },
  ])
})
```

```json filename="Initial Data"
{ "alertVideo": null }
```
